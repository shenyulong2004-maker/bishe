## 登录页 UI 设计稿（实现对照版）

### 视觉风格
- 玻璃拟态 + 渐变背景（低饱和、低对比的背景图/Canvas 作为氛围层）
- 卡片式表单（高对比文本、清晰聚焦态、少装饰线条）
- 强调可读性与键盘交互，不依赖纯色块提示错误

### 配色方案（Design Tokens）
#### 亮色模式
- 背景：radial + linear 渐变
- 表面层：rgba(255,255,255,0.78)
- 文本：#0f172a
- 次级文本：rgba(15,23,42,0.7)
- 品牌色：#2563eb
- 错误色：#b91c1c
- 焦点环：rgba(37,99,235,0.35)

#### 深色模式（prefers-color-scheme: dark）
- 背景：#0b1220 → #0f172a 的渐变 + 低强度 radial 光斑
- 表面层：rgba(2,6,23,0.7)
- 文本：#e2e8f0
- 次级文本：rgba(226,232,240,0.72)
- 品牌色：#60a5fa
- 焦点环：rgba(96,165,250,0.35)

### 布局（响应式）
- 容器：居中，padding 24（移动端 16）
- 表单卡片宽度：min(440px/460px, 100%)
- 移动端（<= 420px）：卡片 padding 18，标题字号略缩小，确保无横向滚动

### 组件与交互
#### 输入框（账号/密码）
- 使用 el-input，支持 clearable / show-password
- 支持 autocomplete（username/current-password）
- 错误提示：输入框下方文本 + 顶部汇总区域（role="alert"）
- 聚焦态：边框变为品牌色 + 4px 焦点环（满足可见焦点要求）

#### 用户类型选择
- 当可登录角色 > 1 时显示
- 选择框与输入框同风格同焦点态

#### 登录按钮
- 全宽按钮，高度 44px，圆角 12px
- loading 状态防止重复提交

### 无障碍（WCAG 2.1 对照）
- 可感知：标签与输入通过 label + input-id 关联；错误信息在 role="alert" 区域可被辅助技术读出
- 可操作：Tab 顺序可达；聚焦态清晰可见；Enter 提交（密码框）
- 可理解：错误提示简短明确（例如“请输入账号/请输入密码/请选择用户类型”）
- 兼容：深色模式通过媒体查询；表单控件对比度在两种模式均保证可读
